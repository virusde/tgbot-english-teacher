# Telegram English Teacher Bot

Telegram-бот для изучения английского с короткими уроками, практикой, разбором ошибок и режимом разбора собственных фраз.

## Что умеет бот

- показывает короткие уроки по темам;
- сразу после урока запускает практику по теме;
- разбирает ошибки после каждого ответа;
- предлагает после практики: повторить тему или перейти дальше;
- показывает слова текущей темы;
- проводит общий квиз по уже изученным словам;
- подсказывает слова, которые стоит повторить;
- хранит прогресс пользователя в локальном JSON;
- умеет разбирать пользовательские фразы и вопросы через `/rules`;
- при наличии OpenAI генерирует новые темы, мягче проверяет ответы и делает разбор фраз подробнее.

## Как проходит обучение

Обычный сценарий такой:

1. Пользователь запускает `/lesson`.
2. Бот либо спрашивает тему, либо сразу берёт уже сохранённую тему.
3. Показывает теорию, примеры и слова.
4. Сразу запускает практику по теме.
5. После завершения показывает разбор ошибок и предлагает:
   - `🔁 Еще практика`
   - `➡️ Следующий урок`

Если OpenAI недоступен, бот продолжает работать на встроенных уроках и локальной логике проверки.

## Команды

- `/start` — приветствие и краткое объяснение режима работы.
- `/help` — список команд и подсказки по использованию.
- `/lesson` — начать следующий урок.
- `/lesson <тема>` — сразу запустить урок на конкретную тему.
- `/topic <тема>` — сохранить тему для следующего урока.
- `/words` — показать слова текущей темы.
- `/quiz` — общий квиз по уже изученным словам.
- `/repeat` — показать слова, в которых было больше ошибок.
- `/progress` — показать прогресс: уроки, правильные ответы, ошибки, точность.
- `/rules` — включить режим разбора своей фразы или вопроса.
- `/rules <фраза>` — сразу разобрать конкретную фразу.

## Логика `/lesson` и `/topic`

Поддерживаются несколько сценариев выбора темы:

- `/lesson`
  Если тема уже была сохранена через `/topic` или обычным сообщением, бот сразу запускает урок по ней.
  Если темы нет, бот спрашивает, какую тему подобрать.

- `/lesson путешествия`
  Бот сразу пытается построить урок на тему `путешествия`.

- `/topic насекомые`
  Бот запоминает тему для следующего урока и показывает кнопку `▶️ Начать урок`.

- Обычное сообщение вроде `Хочу урок про путешествия`
  Бот тоже сохраняет тему и предлагает сразу начать урок кнопкой `▶️ Начать урок`.

Если OpenAI сейчас недоступен, тема всё равно сохраняется, но вместо неё может быть запущен встроенный урок. В этом случае бот предупреждает, что пользовательская тема запомнена и будет использоваться, когда AI снова станет доступен.

## Логика `/rules`

`/rules` нужен для разбора пользовательских фраз и вопросов на английском.

Поддерживаются два режима:

- `/rules`
  Бот включает режим ожидания и просит прислать фразу следующим сообщением.

- `/rules She don't like coffee`
  Бот сразу делает разбор без дополнительного шага.

Что показывает бот:

- исходную фразу;
- более корректный вариант;
- объяснение ошибок;
- короткие правила, которые стоит запомнить.

Если OpenAI недоступен, бот использует локальный fallback-разбор. Он умеет исправлять базовые случаи вроде:

- порядка слов в вопросе;
- `do / does`;
- согласования сказуемого в Present Simple;
- простых ошибок с `am / is / are`;
- очевидных подсказок по времени.

## OpenAI режим

Если в `.env` указан `OPENAI_API_KEY`, бот использует OpenAI для трёх задач:

- генерация новых уроков;
- более гибкая проверка ответов в практике;
- разбор фраз и вопросов в `/rules`.

Основные переменные:

```env
BOT_TOKEN=ваш_токен
OPENAI_API_KEY=ваш_openai_api_key
OPENAI_MODEL=gpt-5-mini
OPENAI_LESSON_MODEL=gpt-5-mini
OPENAI_REVIEW_MODEL=gpt-5-nano
OPENAI_TIMEOUT_SECONDS=90
```

Поведение моделей:

- `OPENAI_LESSON_MODEL` — модель для генерации уроков;
- `OPENAI_REVIEW_MODEL` — модель для проверки коротких ответов и `/rules`;
- `OPENAI_MODEL` — общий fallback, если отдельная lesson-модель не задана.

Если OpenAI временно упирается в rate limit или квоту, бот автоматически уходит в локальный fallback и сообщает об этом пользователю.

## Хранение прогресса

Прогресс хранится в `data/progress.json`.

Для каждого пользователя бот хранит:

- имя;
- индекс встроенного урока;
- текущую и запрошенную тему;
- завершённые уроки;
- историю тем;
- изученные слова;
- статистику правильных и неправильных ответов;
- статистику по отдельным словам;
- активную сессию практики или квиза.

## Запуск

1. Создайте бота через BotFather и получите токен.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env` на основе `.env.example` и заполните нужные переменные.
4. Запустите бота любым удобным способом.

Через обычный Python:

```bash
python main.py
```

Через PowerShell-скрипт:

```powershell
.\run_bot.ps1
```

Через `.cmd`, если PowerShell неудобен:

```cmd
run_bot.cmd
```

Через локальный portable Python:

```powershell
.\tools\python312\python.exe .\main.py
```

`run_bot.ps1` и `run_bot.cmd` сначала пытаются использовать `tools/python312/python.exe`, а затем системный `python`.

## Тесты

Для быстрого регрессионного прогона используйте:

```powershell
.\tools\python312\python.exe -m unittest discover -s tests -v
```

Тесты покрывают:

- встроенные уроки;
- storage и нормализацию пользовательского состояния;
- helper-логику бота;
- локальный разбор в `/rules`;
- поведение `/lesson` и `/topic`.

## Структура проекта

- `main.py` — точка входа.
- `app/bot.py` — Telegram-хендлеры и основная логика диалога.
- `app/content.py` — встроенные уроки, словарь и упражнения.
- `app/ai_tutor.py` — интеграция с OpenAI.
- `app/rules.py` — локальный разбор фраз для `/rules`.
- `app/storage.py` — хранение прогресса.
- `tests/` — unit-тесты.
- `data/progress.json` — создаётся автоматически.
- `tools/python312` — локальный portable Python.
